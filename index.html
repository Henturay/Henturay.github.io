算法题笔记

发表于 2019-07-14 更新于 2019-07-16

之前没有好好做过相关的练习，现在意识到了重要性，想用算法题练习计算机思维。

有过还算不错的小学奥数的训练，所以觉得很多算法题就是小学奥数，所以一直搁置（也有其他很多事情要占据时间），这两者确实很像，比如题目内容比较生活化，一读就明白在讲啥，但是正儿八经做起来却没那么容易。

现在打算好好做算法题，才发现也有很多不一样：比如在小学奥数中，会告诉你一些事实（条件），看上去似乎很散乱，于是当找到突破点时便可以把所有的都联系起来，产生曲径通幽的快感，所以当题做得多时，也许会自然而然产生一些直觉，这是很迷人很好玩的；其实我觉得数学竞赛大概都是这样的过程，只是后面会用到更多的工具，还会用更长和更跳跃的思维把这些事实（条件）栓在一起。而在算法题中，更多的是合理运用数据结构并找到脉络，把问题变为计算机很容易去解决的（重复的递归的…），这样一道算法题便解决了，但是要想解决得好一些，也就是step少一些，需要去看看题目中有没有一些“捷径”简化问题。

对语言的一点看法

发表于 2019-07-14 更新于 2019-07-16

大概是源于大三结束后的那个暑假，抛开词汇量之外，不知道为什么对于英语的使用总…不很顺手，会隔着一些东西，比如很常见的，先得翻译成中文再翻译成英文，如此是很麻烦的一个过程；于是有没有办法跳过去呢？

是在知乎上，知道了维特根斯坦的哲学，不过那个时候更感兴趣的是《你一生的故事》。

最近忘了是由于什么样的原因，在b站看了一位老师介绍维特根斯坦的哲学，大概了解了一些：事情才是客观的，也就是一个物体和它的动作是客观的，而对于同样的这件事情我们用不同的语言有不同的描述，所以有很多不同的映射（当时看板书，分明就是manifold的图册），而这些映射本身（很有可能）是有些许差别的，似乎我们总有一种感觉：
“人有着这种感觉，即他在心里说出的话比他以言词表述的更为清楚。”
所以我看来，没有一种语言是可以完美到描述所有的事情，总有难以描述的地方；对于同样的事物，每种不同的语言各有擅长，所以要选择适合的那一种。

自己以前也有过对于客观事物的映射这样的想法，读到维特根斯坦很多观点时莫名觉得很亲切。

对于一件事情的描述，总能追究到“原子性的概念”上面，然后看看它们在一起发生了什么，然后我们大概就明白了在说什么。

接下来又回到开篇的问题，我认为用顺手英文的办法就是用英文的思维去思考这个世界的运作：
先瞧瞧一个句子——“I like apples”.我认为这句话与“我喜欢苹果”本质上是同一种语言，只不过是不同的记号，不同的发音来表述的；所以多留意一下，当遇见一个句子都是认识的单词，可是连在一起却不懂时，singular point is coming，congratulations…

我不能说明白语言的本性，但我想做一点很trival的推广：在学习数学，学习物理，学习化学，学习编程语言etc时，应该多注意一下隐匿在符号表象后面的它们真正的语言，对此更详细的似乎我还不是太明白，但这真的很好玩。

对计算机和编程的一点看法

发表于 2019-07-14 更新于 2019-07-17

可能我习惯了数学的学习，所以计算机的学习，我觉得要花些许精力和时间在dirty work上面，但这也更接近于现实。

数学与计算机有很多相似的地方，比如重视逻辑，同时Turing为了解决Hilbert的问题发表了论可计算数，同时von Neumann看见来图灵机的物理特性，于是造出了计算机，可以说数学与计算机的渊源非常深了；后来在王子屯Turing与church有一些工作，非常本质，我不太了解，但应该与语言的本质与哥德尔不完备定理有关；正如代数可以几何化，要是逻辑也可以几何化就好了。

后来计算机发展及其飞速，数学也经历了发展：似乎如今没有人可以成为精通数学各个领域，同时我也想是否有人可以把计算机从机器码到汇编高级语言操作系统等等所有的细节都可以解释的很清楚；看起来都是很庞然大物，但数学的学习始终是自下而上的，因为后面的概念，思想很多情况下都依赖于对于最初的概念的理解，但是计算机得自上而下的学习，我想因为计算机更多情况下是工科，工科讲究实用，例如今天来看汇编的使用场所似乎很少，所以干脆不学，尽管要理解一门高级语言的特性到底得明白汇编，但记住规则并用好就能满足大部分场景，看起来没必要了解的太深入。

所以一个善于学习数学的人，我觉得在学习计算机的时候要注意下思维的转变，否则会被很多dirty work困扰，陷入到没必要的陷阱，如果目的是写出不错的项目。

不仅如此，现在高级语言的特性也越来越容易上手，接近人类语言，各种框架也是简化开发，所以可不可以把精力都放在开发上面呢？似乎不可以，因为有时候需要知道些底层东西；可做到全然了解再去用高级语言开发，似乎没人会这么做——底层东西到底学到什么程度比较好呢？我也不知道。至少于我而言这是件很抓狂的事情：不断控制自己的好奇心，把握好“知道”的平衡点。

不过我觉得还是要对计算机整体，发展的历史有个概念，这样自己确实会踏实很多；况且，想知道“计算机的工作原理”是件很有意思的事情，它并不是什么魔法，看着很不可思议但所有东西都是靠人的智慧搭成的，想明白总能明白。

我认为学习编程的办法，最好还是先过一遍基本语法，然后照着写代码，靠项目驱动：有过这样的体验才知道——原来别人已经写好了这么多的组件、小零件可以供我使用（我不用去写jdk…），我需要做的事情是如何把这些组件合理组装起来（遵循高级语言的逻辑），就像是拼乐高一样。同时我认为高级语言一些机制、库这些东西设计得非常合理，这样每个人都可以去制作一些小零件，一旦制作好之后就是取之不尽得，于是每个人都可以把这些零件拿来使用，确实可以做到很多不可思议的事情啊！

于是也可以看出，学习编程最重要的，我认为是编程思想，是如何把一个问题可以变成计算机可以明白使用的——最能锻炼编程思想的，我认为是多做算法训练。

同时还得去知道那些“零件”，知道别人如何使用这些零件，是一件类似体会到画家或者钢琴家的想象力而感到激动的事情；当然这时候还会有一些dirty work：莫名奇妙的文件，配置环境，规定好的操作——倒适当控制自己的“好奇心”，先照着做别问太多为什么，以后的事就不要现在拿来做——但也得知道有这些概念；我认为，未来的社会需要人们对黑箱越来越适应。

对数学的一点想法

发表于 2019-07-07 更新于 2019-07-14

#一点总的感觉：

正如陈省身先生所说：“数学好玩”。

我看来数学改变了人类历史的发展，这毋庸置疑；我想是因为数学总可以提供工具给其他学科，braid group…理论物理中更数不胜数。

所以又有一个很有意思的问题：为什么数学可以为其他学科提供这么多工具？这有点像先有鸡还是先有蛋——是数学影响了我们在发现其他学科时的思维，还是在发现其他学科从表象追寻本质时，最后会落脚到数学上面…但似乎很显然，数学是对规律的抽象进而总结定律推演出真理，伽罗瓦理论，代数结构的发展便是如此；所以我想自然规律甚至是社会规律，都有其定律，于是这些，自然都可以被包含在数学中去（但我认为表象的本源到底是来自于人的思维，数学只是比其他更接近这样的本源，似乎提一下维特根斯坦的哲学比较好）。

所以我认为数学家是这个地球上最可爱的人了，他们像小孩子一样保持着发现的乐趣，又敢于大胆提出疑问，比如质疑第五公设，同时，把这样的思维运用在生活中，能在浮躁的社会发展里看见何为珍贵，变得有所坚守，不至于为其束缚，知道什么是真正的自由；他们往往善良正义，这里不得不一下《love and math》中的一则小故事。

作为pure math本身的发展，很多insights着实吸引人，但是它们太超前了，以至于我们现在的社会还不能完全领悟到ingigshts的意义，于是为了这些而奋斗终生的数学家，他们很难得到他们各自时代的认可，甚至过得比较穷困；小时候我总认为：一个人有着高尚的节操做着高尚的事情，理应得到尊重。但事实却是：得到尊重的往往只有被他们的那个时代的大多数人都认可的人，就像电影明星，歌手，或者一些互联网巨头…数学家的脑子都是extremely brilliant，要是他们愿意可以赚很多钱，这倒是一种英雄主义，不过也憧憬着上个世纪的大环境。

还想说说我对一些创作的看法：我一直觉得人的思维可以分成艺术和科学的——艺术关乎于how I feel，科学在于how it work，所以倒真的很喜欢一些有艺术思维的数学家（科学家），就像是小孩子拿着四驱车，东看看西瞧瞧，pretty cool，然后想办法弄明白怎么跑起来的，怎么跑得这么快，这是个很好玩很踏实也很浪漫的事情。

但感觉学习数学的书籍都很容易劝退，记录一下走马观花看过的书，觉得有意思的话和想法，还有一些好玩的故事，也当作是对自己的鞭策。
